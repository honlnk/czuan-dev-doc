# 开发计划

 项目关于查询英语记录
查询ai讲解和查询词典数据，不应返回布尔值，应当返回查询结果，没有则返回空
关于查询单词翻译接口，移除除了单词之外的选项，因为项目默认只查询基础单词

进一步计划：
* 优化添加单词的接口设计，仅暴露出单词入口，其他均为默认
* 添加英文文档到oss
* 获取文档中的单词去重并单独统计出现频率，生成一个JSON存储的数据库（需要建新表）
思路：单词存入列表 → 查询oss中是否已经有了查询记录 → 有直接用，获取数据，然后添加到word表，没有则执行查询单词，ai讲解单词后，拿到数据再存入word表。
* 逐个单词执行：查单词，ai讲解，添加单词操作。
* 设计查询单词、翻译、词素接口：分页、模糊、id、多表连查
* 设计前端页面：单词表格页，翻译表格页，词素表格页，添加单词表单，添加关系表单，添加词素表单，ai生成内容弹框，添加文档表单，批量处理单词进度页面，单词/文档与oss内容管理页。
* 优化ai讲解逻辑，尝试引入rag
* 新增ai功能，判断出单词中的词素，与翻译执行相同逻辑（库中没有则添加，库中有则直接用），再添加单词时，一并填入，若有来源单词，需要明确标注单词本身，单词的上义词，派生词同样需要做好说明，单词的通用解释和辅助记忆方法需要ai讲解。
* 新增ai功能，可指定ai生成内容部分用特殊xml标签包围，客户端选中指定内容，可要求ai重新生成内容，并加入自己的文字描述需求。
* 定期清洗数据，链接好表中单词词素等信息的依赖关系，整理好单词的在文档中的使用频率，
